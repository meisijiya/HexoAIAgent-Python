## 📖 概述

## 🎯 为什么需要容错处理

### 问题场景

1. **知识库无匹配**：用户问题超出知识库范围
2. **检索结果差**：相似度太低，结果不相关
3. **API 调用失败**：LLM 或 Embedding API 异常
4. **网络问题**：超时、连接失败

### 当前问题

```
用户：介绍一下量子计算
系统：抱歉，知识库中没有找到相关信息。
```

**问题**：生硬、不友好、没有引导

### 优化目标

```
用户：介绍一下量子计算
系统：我在知识库中没有找到关于量子计算的文档。不过我可以：
1. 尝试用我的知识为你介绍一下量子计算的基本概念
2. 帮你搜索一下最新的量子计算资料
3. 你可以尝试换个问法，比如"量子计算是什么"或"量子计算的应用"

你想选择哪种方式？
```

## 🔧 实现方案

### 方案一：引导用户补充

```python
class ErrorHandler:
    """错误处理器"""
    
    async def handle_no_results(self, query: str) -> str:
        """
        处理无检索结果的情况
        
        Args:
            query: 用户查询
        
        Returns:
            str: 引导性回复
        """
        
        # 分析查询类型
        query_type = self._analyze_query_type(query)
        
        # 生成引导性回复
        if query_type == "technical":
            return self._technical_guidance(query)
        elif query_type == "general":
            return self._general_guidance(query)
        else:
            return self._default_guidance(query)
    
    def _technical_guidance(self, query: str) -> str:
        """技术问题引导"""
        return f"""我在知识库中没有找到关于"{query}"的文档。

你可以尝试：
1. **换个问法**：比如把"如何实现X"改成"X的实现步骤是什么"
2. **提供更多细节**：比如你使用的具体技术栈、版本等
3. **搜索相关主题**：输入 `/搜索 {query}` 来查找外部资料

或者，我可以根据我的知识尝试回答，虽然可能不如知识库中的文档准确。"""
    
    def _general_guidance(self, query: str) -> str:
        """通用问题引导"""
        return f"""抱歉，我的知识库中暂时没有关于"{query}"的内容。

我可以帮你：
1. **搜索外部资料**：输入 `/搜索 {query}`
2. **用我的知识回答**：虽然可能不够详细，但可以给你一个基本概念
3. **推荐相关文档**：我可以看看知识库中有没有相关的内容

你想试试哪种方式？"""
    
    def _default_guidance(self, query: str) -> str:
        """默认引导"""
        return f"""我没有找到与"{query}"完全匹配的信息。

建议：
1. 尝试更具体的描述
2. 检查是否有错别字
3. 使用 `/搜索` 命令查找外部资源

需要我尝试用我的知识来回答吗？"""
```

### 方案二：LLM 参考答案

```python
class FallbackAnswerer:
    """备选回答器"""
    
    async def generate_fallback_answer(self, query: str) -> str:
        """
        生成备选答案
        
        当知识库无匹配时，使用 LLM 自身知识回答
        """
        
        prompt = f"""用户问了一个问题，但知识库中没有找到相关文档。

用户问题：{query}

请根据你的知识，给出一个简洁的回答。

要求：
1. 承认这不是来自官方文档
2. 提供基本的概念解释
3. 建议用户查阅官方文档获取准确信息
4. 保持简洁，不超过 200 字

回答："""
        
        messages = [{"role": "user", "content": prompt}]
        
        response = await llm_client.chat(messages)
        return response
```

### 方案三：混合策略

```python
class SmartErrorHandler:
    """智能错误处理器"""
    
    async def handle_no_results(self, query: str, session_id: str) -> AsyncGenerator[str, None]:
        """
        智能处理无检索结果
        
        策略：
        1. 先引导用户补充
        2. 如果用户选择，生成参考答案
        3. 记录用户反馈
        """
        
        # 1. 生成引导性回复
        guidance = await self._generate_guidance(query)
        yield {"type": "content", "content": guidance}
        
        # 2. 提供选项
        yield {
            "type": "options",
            "options": [
                {"label": "用你的知识回答", "value": "fallback"},
                {"label": "搜索外部资料", "value": "search"},
                {"label": "换个问法", "value": "rephrase"}
            ]
        }
    
    async def handle_user_choice(self, choice: str, query: str, session_id: str):
        """处理用户选择"""
        
        if choice == "fallback":
            # 生成参考答案
            answer = await fallback_answerer.generate_fallback_answer(query)
            yield {"type": "content", "content": answer}
        
        elif choice == "search":
            # 触发搜索
            async for chunk in search_agent.process(query, session_id):
                yield chunk
        
        elif choice == "rephrase":
            # 提供改写建议
            suggestions = await self._suggest_rephrasing(query)
            yield {
                "type": "content", 
                "content": f"你可以试试这样问：\n" + "\n".join(f"- {s}" for s in suggestions)
            }
```

## 🛡️ 异常处理

### API 调用异常

```python
class RobustErrorHandler:
    """健壮的错误处理器"""
    
    async def handle_api_error(self, error: Exception, query: str) -> str:
        """
        处理 API 调用异常
        """
        
        if isinstance(error, httpx.TimeoutException):
            return "抱歉，服务响应超时，请稍后再试。"
        
        elif isinstance(error, httpx.HTTPStatusError):
            if error.response.status_code == 429:
                return "服务繁忙，请稍后再试。"
            elif error.response.status_code == 500:
                return "服务内部错误，请稍后再试。"
        
        elif isinstance(error, Exception):
            logger.error(f"未知错误: {error}")
            return "出现了意外错误，请稍后再试。"
    
    async def handle_embedding_error(self, error: Exception, query: str) -> List[SearchResult]:
        """
        处理 Embedding 异常
        
        降级方案：返回空结果，使用 LLM 自身知识
        """
        
        logger.error(f"Embedding 错误: {error}")
        
        # 降级：返回空结果
        return []
    
    async def handle_llm_error(self, error: Exception, query: str) -> str:
        """
        处理 LLM 异常
        
        降级方案：返回预设回复
        """
        
        logger.error(f"LLM 错误: {error}")
        
        # 降级：返回预设回复
        return "抱歉，我暂时无法回答这个问题。请稍后再试，或者尝试换个问法。"
```

### 重试机制

```python
class RetryHandler:
    """重试处理器"""
    
    def __init__(self, max_retries: int = 3, delay: float = 1.0):
        self.max_retries = max_retries
        self.delay = delay
    
    async def retry(self, func, *args, **kwargs):
        """
        重试执行函数
        """
        
        last_error = None
        
        for attempt in range(self.max_retries):
            try:
                return await func(*args, **kwargs)
            except Exception as e:
                last_error = e
                logger.warning(f"尝试 {attempt + 1} 失败: {e}")
                
                if attempt < self.max_retries - 1:
                    await asyncio.sleep(self.delay * (attempt + 1))
        
        raise last_error
```

## 📊 错误监控

### 错误统计

```python
class ErrorMonitor:
    """错误监控器"""
    
    def __init__(self):
        self.error_counts = {}
        self.error_types = {}
    
    def record_error(self, error_type: str, error_message: str):
        """记录错误"""
        
        # 统计错误类型
        self.error_counts[error_type] = self.error_counts.get(error_type, 0) + 1
        
        # 记录错误详情
        if error_type not in self.error_types:
            self.error_types[error_type] = []
        
        self.error_types[error_type].append({
            "message": error_message,
            "timestamp": datetime.now()
        })
    
    def get_error_summary(self) -> Dict:
        """获取错误统计"""
        
        return {
            "total_errors": sum(self.error_counts.values()),
            "error_types": self.error_counts,
            "recent_errors": self._get_recent_errors()
        }
```

## 🎯 最佳实践

### 1. 分层处理

```
第一层：重试机制
    ↓ 失败
第二层：降级方案
    ↓ 失败
第三层：用户友好提示
```

### 2. 错误分类

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 知识库无匹配 | 引导补充 | 提供选项 |
| API 超时 | 重试 | "请稍后再试" |
| API 限流 | 等待重试 | "服务繁忙" |
| 网络异常 | 重试 | "网络不稳定" |

### 3. 用户引导

- 提供多个选项
- 给出具体建议
- 保持友好语气

## 📝 总结

### 关键点

1. **引导补充**：不是简单说"不知道"，而是引导用户
2. **备选答案**：使用 LLM 自身知识作为参考
3. **优雅降级**：API 异常时提供友好提示
4. **错误监控**：记录和分析错误

### 设计亮点

1. **分层处理**：重试 → 降级 → 提示
2. **用户引导**：提供选项，让用户选择
3. **智能回复**：根据问题类型生成不同回复
4. **错误学习**：记录错误，持续优化

### 后续优化
