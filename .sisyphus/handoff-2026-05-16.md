## 🏗️ 架构关键决策

| 决策 | 说明 |
|------|------|
| **ChatAgent 唯一入口** | Orchestrator 已删除，所有 `/api/chat` → ChatAgent → Phase1 LLM 路由 |
| **LLM 路由 > 关键词** | Phase1 prompt 由 SKILLS 动态生成，LLM 返回 JSON `{"route":"...","query":"...","reason":"..."}` |
| **Function Calling > 文本解析** | ReAct v3 用 `chat_with_tools()`，删除 `_parse_response`，不再有格式脆弱性 |
| **content 通道统一** | 思考链和最终答案都走 `content` 事件，`react_formatted` 只带 tools 元数据 |
| **Redis LPUSH 修复** | `rpush` → `lpush`，`ltrim(0,19)` 正确保留最新 20 条 |
| **百度 429 fallback** | 检测 429 → 返回友好提示 → LLM 自行回答 |

## 🔄 ReAct v3 数据流

```
用户消息 → ChatAgent Phase1 分类 → route=react
  ↓
ReActAgent.process():
  ┌─ 第 1 次 LLM 调用（Function Calling）───┐
  │  实时流式 content → 前端消息气泡          │
  │  LLM 决定调用 web_search → tool_calls     │
  │  ✨ yield react_action + react_observation │
  └──────────────────────────────────────────┘
  ┌─ 第 2~N 次 LLM 调用 ────────────────────┐
  │  基于搜索结果继续分析                     │
  │  最多 3 次搜索，max 5 轮迭代              │
  └──────────────────────────────────────────┘
  ┌─ 最终答案 ──────────────────────────────┐
  │  yield react_formatted {thought:"", tools:[...]} │
  │  前端渲染工具来源链接                      │
  └──────────────────────────────────────────┘
```

## 🔌 SSE 事件类型（前端处理）

| 事件 | 含义 | 前端处理 |
|------|------|----------|
| `routing` | Agent 路由结果 | 状态栏显示 |
| `content`（无 event） | 实时流式文本 | 消息气泡逐字 |
| `react_action` | 工具调用通知 | 状态栏 |
| `react_observation` | 工具返回结果 | 黄色观察块 |
| `react_formatted` | 最终元数据（tools+sources） | 来源链接 + 填入消息气泡前 |
| `info` | 友好提示 | 黄色 info 气泡 |
| `sources` / `search_sources` / `knowledge_sources` | 参考来源 | 链接列表 |
| `done` | 对话结束 | session_id 保存 |
